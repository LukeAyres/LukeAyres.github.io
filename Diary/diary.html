Day Two. (10/06/2020)

Forms

To add a form to html we use the form tags:
<form> </form>
Inside a form, input tags can be added to get input from the user and a button tag can be used for the user to send the form:
<form> <input> </input> <button> Submit </button> </form>
Inputs can take the form of radio buttons, check-boxes or single or multi-line text fields. Check it out!

Let's make a basic form that asks for a users name and opinion of our form.
First we'd add the form tag and specify a location where we'd want the information provided by our user to be sent for now we can set the location to a hashtag so that it is not sent anywhere.
Next the input tag will be given a property of type set to "text" which surpriseing enough specifies what type of input to show. We can then give the input placeholder text with the....placeholder property.
Finally we would add the buttons with the relevant input tags and a submit button. The submit button should be given a type of "submit".
The code and resulting html can be seen below.

<form action="#">   <label> Please Enter Your Name:</label>   <input type="text" placeholder="name" required>   <br>   Would you reccomend this form?   <br>   <label>    <input type="radio"> Yes   </label>   <label>    <input type="radio"> No   </label>   <br>   <button type="submit">Submit</button> </form>


Would you recommend this form?


Notice the label and br tags? It is considered best practice to wrap all inputs with such label tags and br tags (breaks) add line space.
Also we can ensure the user provides with data in a certain field before submit by adding the required property. Try clicking submit without providing a name.

There are more properties that should really be added to complete our form... Can you see any problems with our form??
Try looking again. I'll wait.
We can both recommend AND not recommend our form! What is this mayhem.
Well at the moment we haven't connected the radio buttons, as far the form is concerned they could be related to different questions hence it would be very reasonable to be able to select both. Another issue might occur to you: what exactly will be sent if the user opts for "yes" (or "no" 😟 ), we haven't provided the code with any information on what to send for a selected radio button.
We can add a name attribute to each input; all inputs with the same name will be considered part of the same family and so a user will not be able to select more than one radio button. We use the value attribute to tell the form what information to send.

<form action="#">   <label for="username"> Please Enter Your Name:</label>   <input id="username" type="text" name="username" placeholder="name" required>   <br>   Would you reccomend this form?   <br>   <label for="yes">    <id="yes" type="radio" name="recommend" value="yes" checked> Yes   </label>   <label for="no">    <id="yes" type="radio" name="recommend" value="no"> No   </label>   <br>   <button type="submit">Submit</button> </form>


Would you recommend this form?


Try submitting data now and then check out the url at the top of the screen. You should be able to see useful information has now be passed on for example username="Dave" and recommend="yes".
Also you can see how we've added a for tag to our labels and a unique id to the inputs, this associates the two (hover over the labels and see what happens).
We've also added a cheeky checked property can you guess what that does?
